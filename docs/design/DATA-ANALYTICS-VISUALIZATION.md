# FLEXR Data & Analytics System
## Complete Metrics, Graphs, and Progress Visualization

---

# PHILOSOPHY

> **"If you can't measure it, you can't improve it."**

FLEXR captures EVERYTHING and shows it ALL. Users who love data get:
- Every metric tracked
- Every trend visualized
- Every improvement quantified
- Every comparison available

**No hiding data. No dumbed-down views. Full transparency.**

# PART 1: ALL METRICS WE TRACK

## 1.1 Running Metrics

### Per-Session Running Data
| Metric | Unit | Description |
|--------|------|-------------|
| **Distance** | km/m | Total running distance |
| **Duration** | min:sec | Total running time |
| **Average Pace** | min/km | Overall pace |
| **Best Pace** | min/km | Fastest split |
| **Splits** | min/km per 200m/400m/km | Pace per segment |
| **Cadence** | steps/min | Running cadence |
| **Stride Length** | meters | Calculated from cadence + pace |
| **Elevation Gain** | meters | Total climb |
| **Elevation Loss** | meters | Total descent |
| **Ground Contact Time** | ms | If available from watch |
| **Vertical Oscillation** | cm | If available from watch |

### Heart Rate Running Data
| Metric | Unit | Description |
|--------|------|-------------|
| **Average HR** | bpm | Mean heart rate |
| **Max HR** | bpm | Peak heart rate |
| **Min HR** | bpm | Lowest during run |
| **HR at Start** | bpm | Beginning of run segment |
| **HR at End** | bpm | End of run segment |
| **Time in Zone 1** | min:sec | Recovery (<60% max) |
| **Time in Zone 2** | min:sec | Aerobic (60-70%) |
| **Time in Zone 3** | min:sec | Tempo (70-80%) |
| **Time in Zone 4** | min:sec | Threshold (80-90%) |
| **Time in Zone 5** | min:sec | VO2max (90-100%) |
| **HR Drift** | % | HR increase at same pace over time |
| **Cardiac Decoupling** | % | Efficiency loss indicator |

### Compromised Running Specific
| Metric | Unit | Description |
|--------|------|-------------|
| **Fresh Pace** | min/km | Baseline pace (rested) |
| **Compromised Pace** | min/km | Pace after station |
| **Pace Degradation** | sec/km | Difference from fresh |
| **Degradation %** | % | Percentage slower |
| **Previous Station** | type | What station preceded |
| **Recovery Time** | sec | Time to return to target pace |
| **Post-Station HR** | bpm | Starting HR for run |

# SUMMARY

## Data Philosophy

1. **Track everything** - Every metric that matters
2. **Show everything** - No hidden data
3. **Visualize clearly** - Graphs for visual learners
4. **Numbers for nerds** - Exact values always available
5. **Trends over time** - Progress is the goal
6. **Isolate what matters** - Run data separate from station data
7. **Insights, not just data** - AI explains what it means
8. **Export freely** - Your data is yours

## Key Dashboards

| Dashboard | Purpose |
|-----------|---------|
| Main | Daily readiness, quick stats |
| Running | Pace zones, compromised analysis, trends |
| Stations | PR tracking, weakness/strength, distribution |
| Heart Rate | Zones, efficiency, recovery |
| Recovery | HRV, sleep, training load |
| Progress | Long-term trends, predictions, PRs |

## Unique FLEXR Data

What no other app shows:
- **Compromised running pace per station**
- **Degradation % trends**
- **Post-station recovery patterns**
- **Running isolated from stations**
- **Station impact on running performance**
